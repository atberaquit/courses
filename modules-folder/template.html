Mastering the Terminal and Git Intro

Objectives:

  • Understand the basics of the Command Line Interface (CLI)
  • Learn essential CLI commands for file and directory management
  • Familiarize with Git commands for version control and collaboration

Lesson Proper:

Introduction:

Start by explaining the importance of the Command Line Interface (CLI) and Git for web development projects. Discuss the benefits of using the CLI and Git for efficient development and collaboration. Highlight the relevance of mastering CLI and Git commands for web developers.

Command Line Interface (CLI) Commands:
Introduction to the CLI:
  • Define the Command Line Interface (CLI) and its role in interacting with the operating system.
  • Explain the advantages of using the CLI over graphical user interfaces for certain tasks.
  • Discuss common CLI applications such as Terminal (macOS and Linux) and Command Prompt (Windows).
Basic CLI Commands:
  • Introduce essential CLI commands for file and directory management:
  • pwd: Print the current working directory.
  • ls: List files and directories in the current directory.
  • cd: Change the current directory.
  • mkdir: Create a new directory.
  • touch: Create a new file.
  • rm: Remove files and directories.
  • cp: Copy files and directories.
  • mv: Move or rename files and directories.
Git Commands for Version Control:
Introduction to Git:
  • Explain the concept of version control and its importance in software development.
  • Introduce Git as a widely-used distributed version control system.
  • Highlight the benefits of using Git for tracking changes and collaborating on projects.
Basic Git Commands:
  • Discuss essential Git commands for version control:
  • git init: Initialize a new Git repository.
  • git clone: Clone a remote repository to the local machine.
  • git add: Stage changes for commit.
  • git commit: Commit changes with a descriptive message.
  • git push: Push local changes to a remote repository.
  • git pull: Fetch and merge changes from a remote repository.
  • git status: Check the current status of the repository.
  • git log: View commit history.
Activity:

Provide a set of exercises for students to practice using CLI and Git commands. Assign tasks such as creating directories, navigating the file system, creating and modifying files, initializing a Git repository, making commits, and pushing changes to a remote repository. Encourage students to explore additional CLI and Git commands beyond the basics covered in the lesson.
